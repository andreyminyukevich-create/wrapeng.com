# Telegram Lead Parser

Парсер Telegram-постов по ключевым словам для поиска лидов на детейлинг-услуги.
Находит посты через глобальный поиск Telegram и отправляет карточки лидов в указанный чат через бота.

## Как это работает

1. Раз в N секунд (по умолчанию 30 минут) проходит по списку ключевых слов
2. По каждому слову делает глобальный поиск через Telegram API
3. Новые посты форматирует в карточку лида и отправляет через бота
4. Запоминает уже отправленные посты, чтобы не дублировать

## Настройка

### 1. Получить API_ID и API_HASH

1. Зайди на [my.telegram.org](https://my.telegram.org)
2. Авторизуйся по номеру телефона
3. Перейди в **API development tools**
4. Создай приложение (название любое)
5. Скопируй **App api_id** и **App api_hash**

### 2. Создать бота и получить BOT_TOKEN

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Отправь `/newbot`
3. Придумай имя и username бота
4. Скопируй токен вида `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`

### 3. Узнать MY_CHAT_ID

1. Открой [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправь `/start`
3. Бот ответит твоим ID (числом)

> Можно отправлять в личку боту (тогда MY_CHAT_ID = твой ID) или в группу
> (добавь бота в группу, MY_CHAT_ID = ID группы, будет отрицательным числом).

### 4. Сгенерировать SESSION_STRING

Локально на своём компьютере:

```bash
pip install telethon
python generate_session.py
```

Скрипт попросит ввести API_ID, API_HASH, номер телефона и код подтверждения.
После авторизации выведет строку сессии — скопируй её для переменной `SESSION_STRING`.

> **Важно:** Строка сессии — это доступ к твоему аккаунту Telegram. Храни её как пароль.

## Env-переменные

| Переменная        | Обязательная | Описание                                  | По умолчанию |
| ----------------- | ------------ | ----------------------------------------- | ------------ |
| `API_ID`          | да           | Telegram API ID (число)                   | —            |
| `API_HASH`        | да           | Telegram API Hash                         | —            |
| `BOT_TOKEN`       | да           | Токен бота от BotFather                   | —            |
| `MY_CHAT_ID`      | да           | ID чата куда слать лиды (число)           | —            |
| `SESSION_STRING`  | да           | Telethon StringSession                    | —            |
| `SEARCH_INTERVAL` | нет          | Интервал между проходами в секундах       | `1800`       |
| `DATA_DIR`        | нет          | Путь для хранения seen_ids.json           | `/data`      |

## Деплой на Railway

### 1. Создать проект

1. Зайди на [railway.app](https://railway.app) и создай новый проект
2. Подключи GitHub-репозиторий с этим кодом

### 2. Подключить Volume

1. В проекте Railway нажми **+ New** → **Volume**
2. Укажи Mount Path: `/data`
3. Привяжи к сервису парсера

Volume нужен, чтобы файл `seen_ids.json` (список уже отправленных постов) сохранялся между редеплоями.

### 3. Добавить переменные окружения

В настройках сервиса (Variables) добавь все обязательные переменные:
- `API_ID`
- `API_HASH`
- `BOT_TOKEN`
- `MY_CHAT_ID`
- `SESSION_STRING`

### 4. Запуск

Railway автоматически задеплоит сервис. В логах должно появиться:
```
Парсер запущен, слежу за 16 ключевыми словами
```

Бот пришлёт это же сообщение в указанный чат.

## Локальный запуск

```bash
pip install -r requirements.txt

export API_ID=12345
export API_HASH=your_hash
export BOT_TOKEN=your_bot_token
export MY_CHAT_ID=your_chat_id
export SESSION_STRING=your_session_string
export DATA_DIR=./data

python parser.py
```
