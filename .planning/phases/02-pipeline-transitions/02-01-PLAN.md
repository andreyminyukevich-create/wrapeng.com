`injectOutsource()`: getElementById check, insertAdjacentHTML, bind cancel/save/overlay-click.

`gatherOutsourceForm()`: Returns `{ contractor_name, work_type, deadline, outsource_type, notes }`. Use `.value.trim() || null` for text fields, `.value || null` for date.

`openModalOutsource(calcId, carName)`: inject, set _outsourceCalcId, set carName via .textContent, reset all fields (contractor='', workType='', deadline='', type selectedIndex=0, notes=''), show modal.

`saveOutsource()`: Two-step pattern:
1. `const btn = document.getElementById('btnOutsourceSave'); btn.disabled = true; const origText = btn.textContent; btn.textContent = 'Сохранение...';`
2. INSERT into `outsource_records`: `await window._sb.from('outsource_records').insert({ calc_id: _outsourceCalcId, studio_id: window._boardCtx.studioId, ...gatherOutsourceForm() })`
3. On insert error: `console.error('[pipeline] outsource insert:', insertErr); window._showToast('error', 'Ошибка сохранения'); return;`
4. PATCH status: `const ok = await window._updateStatus(_outsourceCalcId, 'outsourced');`
5. On success: `window._closeModal('modalOutsource'); window._showToast('success', 'Отправлено на аутсорсинг'); window._loadBoard();`
6. Finally block: restore btn.disabled and btn.textContent.

`openModalReturn(calcId, carName)`: inject, set _returnCalcId, carName via .textContent, set returnDate to today (`new Date().toISOString().slice(0, 10)`), clear returnNotes, show modal.

`saveReturn()`: UPDATE pattern (not INSERT):
1. Button lock (same pattern).
2. Gather: `const returnDate = document.getElementById('returnDate').value || new Date().toISOString(); const returnNotes = document.getElementById('returnNotes').value.trim() || null;`
3. UPDATE outsource_records: `await window._sb.from('outsource_records').update({ returned_at: returnDate, return_notes: returnNotes }).eq('calc_id', _returnCalcId).eq('studio_id', window._boardCtx.studioId).is('returned_at', null);`
   - Uses `.is('returned_at', null)` to target only the active (unreturned) record per Pitfall 3 in research.
4. On error: toast error, return.
5. PATCH status: `await window._updateStatus(_returnCalcId, 'in_progress');`
6. On success: close modal, toast 'Авто возвращено в работу', loadBoard.
